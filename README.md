# MindboxAreaCalc

Тестовое задание для компании MindBox

API проектировался исходя из следующих целей:
1. При добавлении разработчиком библиотеки новых фигур не должна требоваться перекомпиляция
   внешнего клиента (пользователя библиотеки)
2. Разработчик внешнего клиента может сам добавить свою формулу площади фигуры - отнаследоваться от 
   базового класса, реализовать нужный интерфейс, расширить class factory и пользоваться.
3. Если разработчик внешнего клиента сделал п.2, а потом разработчик библиотеки добавил формулу площади 
   новой фигуры, функциональности не должны конфликтовать. По этой причине в решении не используется enum "Тип фигуры". 
   Тип фигуры - это строка. Чтобы не заморачиватся со строковыми константами, используется название класса
